AgentPay Documentation

AgentPay is the first ERC-8183 powered API marketplace for AI agents. Every API call is a structured on-chain job — work defined, USDC escrowed, deliverable verified, payment released trustlessly on Arc.

⚡

Per-Request Billing

Each API call costs 0.000001 USDC. Pay exactly for what you use.

🔒

Replay Protection

Every payment ID is single-use. No double-spending, no replay attacks.

🤖

Agent-Native

Designed for autonomous AI agents that transact without human approval.

🌐

On-Chain Settlement

Payments verified on ARC L1 via Circle's Programmable Wallets API.

Problem & Solution

The Problem

Today's API economy is built for humans. Subscriptions, OAuth flows, and credit card billing assume a person is on the other end. As AI agents become autonomous economic actors, they need a payment primitive that works without human intervention.

  • Subscription models — waste money when agents run infrequently
  • API keys — no payment enforcement, easy to abuse
  • Credit cards — require human authorization flows
  • Traditional crypto — high gas fees make micropayments impractical

The Solution

AgentPay implements ERC-8183: a standard for on-chain AI agent jobs. Every API call is a structured job — posted on-chain, accepted by a provider, executed, verified, and paid trustlessly via USDC escrow on ARC. Zero subscriptions. Zero trust required.

ERC-8183 Job Lifecycle

1

Initiate Transfer

Agent calls POST /api/pay/transfer. Server creates a Circle USDC transfer on ARC Testnet.

2

Poll for Confirmation

Agent polls GET /api/pay/status/:id until state reaches COMPLETE. Takes 3–8 seconds.

3

Present Proof & Receive Data

Agent calls gated endpoint with X-Payment-Id header. Middleware verifies and responds.

API Endpoints

Base URL: https://agentpay-production-7a2a.up.railway.app

POST/api/job/create
Posts a new ERC-8183 job on-chain. Escrows USDC and returns a jobId.
GET/api/job/status/:jobId
Polls job lifecycle state. Wait for state: "ACCEPTED" then "COMPLETED".
GET/api/weather?city=london
Current weather. Requires X-Job-Id. Cities: new-york, london, tokyo, sydney.
GET/api/crypto?coin=bitcoin
Crypto price via CoinGecko. Requires X-Job-Id.
GET/api/exchange?from=usd-coin&to=usd
Exchange rate between two assets. Requires X-Job-Id.

Authentication

All gated endpoints require a valid Circle transfer ID as HTTP header:

X-Job-Id: a1b2c3d4-e5f6-7890-abcd-ef1234567890

Each job ID can only be used once. Reuse returns 402 Job Already Settled.

Error Codes

402 Payment Required — No X-Job-Id header.

402 Payment Already Used — Transfer ID already spent.

402 Invalid Payment — Transfer not COMPLETE or wrong amount.

404 Not Found — Resource not found.

500 Internal Server Error — Unexpected error.
